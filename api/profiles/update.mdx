## Endpoint

```
PUT /api/profiles/:id
```

## Description

Updates the profile information for a specific user. Users can update their own profiles, while administrators can update any profile.

See implementation in `server/src/modules/profiles/profiles.controller.ts:42-57`.

## Authentication

<Info>
  This endpoint requires Bearer token authentication. Include your access token in the `Authorization` header.
</Info>

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user whose profile you want to update.
  
  **Example**: `abc123`
</ParamField>

## Request Body

All fields are optional. Include only the fields you want to update.

<ParamField body="email" type="string" optional>
  User's email address
  
  **Example**: `newemail@example.com`
</ParamField>

<ParamField body="full_name" type="string" optional>
  User's full name
  
  **Example**: `Dr. Jane Smith`
</ParamField>

<ParamField body="role" type="string" optional>
  User role: `USER` or `ADMIN`
  
  <Warning>Only administrators can change user roles</Warning>
</ParamField>

<ParamField body="is_active" type="boolean" optional>
  Whether the user account is active
  
  **Example**: `true`
</ParamField>

<ParamField body="birth_date" type="string" optional>
  User's date of birth in ISO 8601 format (YYYY-MM-DD)
  
  **Example**: `1985-03-15`
</ParamField>

<ParamField body="is_profile_complete" type="boolean" optional>
  Whether the user has completed their profile setup
  
  **Example**: `true`
</ParamField>

### Example Request Body

```json
{
  "full_name": "Dr. Jane Smith",
  "birth_date": "1985-03-15",
  "is_profile_complete": true
}
```

## Error Responses

### 400 Bad Request

```json
{
  "statusCode": 400,
  "message": [
    "email must be an email",
    "birth_date must be a valid ISO 8601 date string"
  ],
  "error": "Bad Request"
}
```

Invalid data provided in request body.

### 401 Unauthorized

```json
{
  "statusCode": 401,
  "message": "Token inválido o expirado",
  "error": "Unauthorized"
}
```

Missing or invalid Bearer token.

### 404 Not Found

```json
{
  "statusCode": 404,
  "message": "Perfil no encontrado.",
  "error": "Not Found"
}
```

No profile exists for the specified user ID.

## Use Cases

- User completing their profile during onboarding
- Updating contact information
- Admin changing user roles or status
- Marking profile as complete after required fields filled
- Deactivating user accounts

## Validation Rules

From `server/src/modules/profiles/dto/update-profile.dto.ts`:

- `email` must be a valid email format
- `birth_date` must be a valid ISO 8601 date string (YYYY-MM-DD)
- `is_active` must be a boolean
- `is_profile_complete` must be a boolean
- `full_name` must be a string
- `role` must be either `USER` or `ADMIN`
