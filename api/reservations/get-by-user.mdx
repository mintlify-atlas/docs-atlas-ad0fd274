## Endpoint

```
GET /api/reservations/user/:client_id
```

## Description

Retrieves all reservations associated with a specific user by their client ID.

See implementation in `server/src/modules/reservations/reservations.controller.ts:89-107`.

## Authentication

<Info>
  This endpoint requires Bearer token authentication. Include your access token in the `Authorization` header.
</Info>

## Path Parameters

<ParamField path="client_id" type="string" required>
  The unique identifier of the client whose reservations you want to retrieve
  
  Example: `abc123`
</ParamField>

## Response

<ResponseField name="status" type="number">
  HTTP status code (200 for success)
</ResponseField>

<ResponseField name="message" type="string">
  Success message including the client ID
</ResponseField>

<ResponseField name="data" type="array">
  Array of reservation objects for the specified user
  
  <Expandable title="Reservation object">
    <ResponseField name="id" type="string">
      Unique identifier for the reservation
    </ResponseField>
    
    <ResponseField name="client_id" type="string">
      ID of the client (matches the path parameter)
    </ResponseField>
    
    <ResponseField name="professional_id" type="string">
      ID of the professional
    </ResponseField>
    
    <ResponseField name="start_time" type="string">
      Start time in ISO 8601 format
    </ResponseField>
    
    <ResponseField name="end_time" type="string">
      End time in ISO 8601 format
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Reservation status: `PENDING`, `CONFIRMED`, or `CANCELLED`
    </ResponseField>
    
    <ResponseField name="session_modality" type="string">
      Session type: `Virtual`, `Presencial`, or empty string
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      Creation timestamp in ISO 8601 format
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET https://your-domain.com/api/reservations/user/abc123 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

```typescript TypeScript
const getUserReservations = async (clientId: string) => {
  const response = await fetch(`https://your-domain.com/api/reservations/user/${clientId}`, {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json'
    }
  });
  
  const data = await response.json();
  return data;
};

// Usage
const reservations = await getUserReservations('abc123');
```

</CodeGroup>

## Use Cases

- Display a user's upcoming appointments
- Show reservation history
- Allow users to manage their bookings
- Build a user dashboard with reservation information

## Notes

- Returns an empty array if the user has no reservations
- All reservations for the user are returned regardless of status
- To get reservations for a professional, use [Get By Professional](/api/reservations/get-by-professional)
